В данном модуле реализована процедурная генерация болезней, симптомов болезней, а также медикаментов для лечения болезни.
Внутри решения вложены 3 текстовых файла с ассетами для процедурной генерации: list_name_illnesses.txt - файл с названиями болезней,
list_name_medicines.txt - файл с названиями медикаментов, list_name_symptoms.txt - файл с названиями симптомов. Все названия ассетов, кроме симптомов, являются вымышленными.
По желанию их можно легко дополнить или заменить своими.

В решении реализованы 3 самостоятельных класса: illness - болезнь, symptoms - симптомы, medicines - медикаменты.
В каждом из данных классов реализованы методы класса get_all_posible_name(cls) и init_group(cls, number).
Метод get_all_posible_name выполняет функцию чтения данных из файла ассетов и выполняет подготовку данных для дальнейшей работы.
Метод init_group(cls, number) выполняет функцию инициализации (создания) экземпляров в зависимости от количества требуемых объектов. Имена для создания экземпляров метод выбирает
случайным образом из массива подготовленных ассетов. При создании каждого из экземпляров он добавляется в список созданных экземпляров, который находится внутри класса.

Экземпляры симптомов и медикаментов связаны с экземплярами болезней через агрегирование. Тоесть, у каждого экземпляра болезни есть список экземпляров сиптомов и список списков
экземпляров медикаментов.

Существует возможность задать какому-либо медикаменту несовместимый медикамент.

В классе illness реализован метод класса output_info_class(cls), который выводит информацию в консоль о названии болезни, ее симптомах, и способах лечения.
В классе medicines данный метод выводит название медикамента и название несовместимого медикамента (если такой имеется).

Так же реализованы некоторые вспомогательные методы для получения некоторых данных или задания вручную/добавления данных, получения всех совместимых медикаментов, получения/добавления несовместимых медикаментов.

Создание экземпляров, процедурная генерация, объединение и проверка происходит в файле main.py. Все классы импортируются в данный файл.

Генерация происходит следующим образом: в условном мире создается n-случайных экземпляров симптомов из файла ассетов. Далее создаются все возможные неповторяющиеся комбинации данных симптомов.
Затем создаются n-случайных экземпляров медикаментов из файла ассетов. Можно некоторому количеству созданных медикаментов добавить несовместимый медикамент. Создаются все возможные неповторяющиеся комбинации данных медикаментов не учитывая несовместимости. Далее таким же образом создаются
n-случайных экземпляров болезней из файла ассетов. Выполняются проверки на возможность заполнения всех экземпляров болезней симптомами, медикаментами. Происходит добавление случайной комбинации симптомов,
медикаментов к экземпляру болезни. При добавлении случайных комбинаций медикаментов происходит удаление некорректных лечений.

Данную генерацию можно использовать, например, в некоторых игровых проектах. Это позволит добавить в проект такую "фичу" как реиграбильность, так как будут созданы совершенно новые болезни, симптомы и способы их лечения.

Данная процедурная генерация является просто примером. Ее можно доработать под свои нужды каждому. Например, добавить в класс болезнь описание этой болезни. Для этого придется придумать каждой болезни из ассета свое описание и при генерации вставить это описание непосредственно в класс.
Точно также с медикаментами и симптомами, можно дать каждому уникальное описание и встроить в класс.
Можно также переделать механизм задания несовместимостей между медикаментами. Например, все медикаменты разделить по группам, и создавать несовместимости по группам, генерируя вначале случайную/случайные несовместимые группы.
